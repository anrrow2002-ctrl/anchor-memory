# Anchor Memory 0.9.13 升级说明

本版保留 0.9.12 的归档全量整理流程，修复逐楼摘要反复提示“为空或过短”的问题。

## 修复内容

1. 校验六个 Godlog XML 字段是否完整。
2. 单独校验 Cond 长度：长楼层至少 200 字，短楼层按原文动态设置下限。
3. 首次输出为空、过短或缺字段时，自动追加一次带具体原因的纠正重写。
4. 支持更多 OpenAI-compatible 返回结构和转义 XML。

安装时继续覆盖原目录 `anchor-memory-0.9.11-REWRITE-CONTROLS`，不要保留两个副本。聊天数据结构未改变，原有摘要、锚点、合并、关系表和档案不会被清空。
