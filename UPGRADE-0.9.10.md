# Anchor Memory 0.9.10 升级说明

直接覆盖安装即可，不需要删除旧插件数据，也不要清空聊天 metadata。

## 本次修复

### 1. 新楼摘要未完成时，关系表不再被误判删除

旧版在摘要 API 返回时，如果 SillyTavern 恰好短暂替换了该消息对象，可能暂时找不到原稳定键，继而触发“摘要生成期间源楼层被删除”，回滚关系表并把人物/物品/场景索引标记为待重建。

0.9.10 改为：

- 先按插件稳定键定位；
- 再按正文签名、发送时间、AI 回合序号和楼层位置重新定位；
- 仍未定位到时进入 8 秒延迟确认，不立即执行删除回滚；
- 已经生成出的摘要候选会暂存，源楼恢复后直接提交，不再次调用摘要 API；
- 只有延迟确认结束后仍确实不存在，才按真实删楼处理。

### 2. 最新待摘要楼层不再阻塞整张关系表

安全重建现在只把“已完成历史中间缺一楼、后面又已有完成摘要”的情况视为阻塞缺口。最新连续若干楼尚在生成摘要时，插件可以先按此前完整历史恢复关系/人物/物品/场景表；新摘要完成后再增量补入。

人物关系待重建期间，主模型仍会收到回退后的最近有效安全快照，而不是整段显示“本次不注入”。人物动态、出场人物库和重要物品表也会检查自身最近同步楼层：只要没有历史中间缺口，且旧表没有超出最近有效摘要，就继续作为安全快照注入；真实删楼/重跑导致边界无法确认时才暂停这些累计表。

### 3. 累计合并恢复连续运行

- 旧安装若仍保存内置默认值 `100`，升级时会一次性调整为 `75`；用户之后可以手动改回。
- 后台一次发现多个已到期边界时，会连续补齐，最多每轮排空 6 个边界；调整间隔触发的追赶最多 12 个边界。
- 状态栏新增“本周期 X/75”，可以直接看距离下一次累计合并还差多少个有效 AI 回合。

注意：若聊天目前只有 104 个有效 AI 回合，按 75 间隔本来就只应有 1 次累计合并；第 2 次在第 150 个有效 AI 回合触发。修复的是“已经越过 150/225 等边界后仍只追赶一次”的队列问题。

### 4. 全量合并不再写同一天流水账

默认规则要求同一天、同一目标/冲突连续推进的场景合成一条因果链。只有日期、核心矛盾、行动目标或关系阶段实质改变时才拆条。旧历史会进一步压缩，本周期新增保留更完整的转折、关键原话、伏笔与道具。

已经保存的旧合并不会在升级瞬间自动花费 API 重写。打开工作台点击“全量合并/重写”：若有新增摘要则提前合并；若没有新增摘要，则按新规则重写最近一次累计锚点，编号和覆盖范围不变，失败时仍保留旧正文。
