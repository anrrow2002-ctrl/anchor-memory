# Anchor Memory 0.9.14 全链路审计

## 审计范围

本次不是在 0.9.13 上追加单点补丁，而是从配置输入、网络请求、响应解析、摘要生成、后台队列、记忆派生、主模型注入、楼层隐藏、逐楼摘要隐藏、归档与转档、聊天切换和暂停恢复逐条复核。

## 主要根因与修复

### 1. “副 API 没有返回摘要正文”误报

旧逻辑会把多种完全不同的失败压缩成一个空文本结果，包括 HTTP 错误、鉴权失败、后端 error 对象、HTML 代理页、未识别响应结构以及仅返回 reasoning 的模型。0.9.14 改为先识别传输层和后端错误，再解析正文；只有取得真实模型文本后才进行 Godlog 格式校验。

### 2. 手机端模型列表拉取

旧逻辑可能在移动端按钮点击时继续读取上一次保存的设置，而不是当前输入框；旧模型也没有在请求开始时清空。0.9.14 在点击时同步地址和密钥、立即清空模型栏和缓存列表，优先使用宿主后端代理，失败后再浏览器直连，并加入请求预算、单次超时、按钮锁和可见错误链。

### 3. 摘要重试与后台队列

网络/配置失败不再触发“纠正摘要”请求。地址、密钥或模型缺少任一项时，后台摘要、锚点和合并队列不启动；设置补全后再恢复待处理任务。已有成功摘要仍采用事务保护，失败重跑不会覆盖旧摘要。

### 4. 归档与转档

转档副本只包含最终累计全量记忆与人物关系、人物、物品、场景等状态，不携带旧逐楼摘要和旧分段锚点。原聊天不能加载来源为自己的转档副本，从入口和执行函数双重阻止详细数据被精简副本覆盖。

### 5. 隐藏楼层所有权

旧逻辑可能仅凭插件 metadata 判断某个已隐藏楼层属于 Anchor Memory，导致玩家手动隐藏的消息在暂停或窗口变化时被错误恢复。0.9.14 只对带明确 Anchor Memory 隐藏所有权字段的楼层执行自动恢复，并在恢复后清理所有权字段；玩家、宿主和其他扩展的隐藏状态保持不变。

### 6. 隐藏逐楼摘要

Godlog 只保存在聊天 metadata，并通过工作台 UI 展示，不写入消息正文。正文清理覆盖插件管理的隐藏 system 行，防止历史 Godlog 泄漏到主模型；摘要任务失败与旧正文可见性相互独立，不会因摘要缺失把超过原文窗口的旧正文重新放回提示词。

## 数据兼容

本版不提升聊天记忆数据版本，不清空现有 Godlog、分段锚点、累计合并、关系表、人物、物品、场景、档案或向量引用。覆盖安装时必须保留原目录名，避免 SillyTavern 把新旧副本同时加载。
