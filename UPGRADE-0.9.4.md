# 升级到 0.9.4 Summary Lock

## 数据迁移

0.9.4 的数据版本为 11，源楼层身份架构版本为 4。首次打开已有聊天时，插件会：

1. 为每条叙事消息建立插件自有稳定消息键；
2. 按原楼层、人物名和时间兼容匹配已有 Godlog；
3. 同步迁移锚点、累计合并、人物关系历史、状态索引键和楼层注入记录；
4. 保留所有已经生成的摘要正文，不会因迁移自动调用副 API 重写。

## 新的摘要规则

- **首次尚未生成：** 自动排队生成；
- **首次生成期间正文仍在变化：** 等待稳定后再生成；
- **已经生成完成：** 永久按已保存摘要处理，不因注入、渲染或宿主元数据变化自动重跑；
- **需要更新：** 用户点击“重跑本楼摘要”，或在摘要编辑框修改后保存。

当已保存摘要对应的原始正文与当前楼不同，楼层卡片仍显示“已保存”，展开后会看到差异提示。该提示不代表摘要丢失，也不会阻塞锚点。

覆盖升级前仍建议在“工具”页导出一次当前记忆 JSON。
