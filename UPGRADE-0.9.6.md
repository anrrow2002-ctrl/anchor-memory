# Anchor Memory 0.9.6 升级说明

## 修复内容

0.9.5 中，关键词兜底通常已经在最终请求构造时写入，但工作台刷新较晚，会造成“模型先跑、召回后出现”的错觉。更重要的是，远程 Embedding 预取若来不及完成，本轮只能使用关键词结果，后到的语义结果不会进入已经发出的请求。

0.9.6 将召回链路改成严格的“发送前完成”：

1. 用户消息写入后启动预取；
2. 最终 Prompt Ready 阶段等待语义结果，最长 1800ms；
3. 超时就在请求发送前使用关键词兜底；
4. 写入最终消息数组后标记“已在主请求发送前注入”；
5. 后到的语义结果只缓存并注明未用于本轮；
6. 面板预览、公开 Preview API、Prompt Inspector dry-run 均不再改写正式注入记录。

## 安装

完整覆盖旧插件目录并刷新 SillyTavern。数据版本仍为 11，不需要重建逐楼摘要、锚点、人物关系或状态索引。
