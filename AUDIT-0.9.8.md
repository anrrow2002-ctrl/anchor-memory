# Anchor Memory 0.9.8 记忆断层审计

## 审计结论

正常摘要全部成功时，15/75 分层在 1–90 回合模拟中没有出现来源断层。

但 0.9.7 存在两个真实连续性问题：第一次锚点形成前，完整逐楼摘要可能超过第六段 1700 Token 上限并被头尾裁剪；此外，任意旧楼摘要失败并离开最近 3 回合原文窗口后，该楼在最终请求中没有事实来源；同时它会阻塞后续分段锚点，使未锚定摘要越来越长。对照模拟中，第 39 回合摘要永久失败时，第 42 回合首次出现缺口。

## 修复

- 未锚定 Godlog 改为按当前数量动态分配长度的紧凑逐回合索引，并把第六段上限提高到 3000 Token、最低保障提高到 1400 Token。
- 增加缺失楼层的受限原文保底材料。
- 保底材料只在该楼离开最近原文窗口后启用。
- 分段锚点与累计合并均能识别保底材料。
- 锚点、合并记录新增 `rawFallbackKeys`，便于审计哪些楼层使用过保底原文。
- 正常 Godlog 仍保持失败/待补写状态，后续可以继续重跑，不会被伪装为完成摘要。
- 主聊天历史仍严格裁剪，不会因为一楼失败而整段恢复旧正文。
