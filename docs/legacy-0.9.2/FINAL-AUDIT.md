# Anchor Memory 0.9.2 FINAL 审计结论

## 一、结论

本包是在 0.9.1 数据流重构基础上的确定版。代码级检查确认并修复了会随着聊天增长逐渐放大的问题，以及若干不会立即报错、但会造成记忆漂移或面板误导的隐性问题。

本次验证包含：JavaScript 语法检查、静态回归断言、设置页 ID/标签对应检查、中文关键词召回行为模型、1500 条合成摘要扫描测试、压缩包完整性检查。容器内无法启动用户本机的 SillyTavern 浏览器环境，因此各 SillyTavern 分支版本和第三方扩展组合仍应在本机做一次真实生成验证。

## 二、最终数据流

1. 每个有效 AI 回合稳定后，由副 API 生成一条 Godlog。
2. 每达到“分段锚点间隔”，把连续未锚定 Godlog 生成分段锚点。
3. 每达到“累计合并间隔”，把本周期分段锚点和未被分段锚点覆盖的 Godlog 合并为新的累计历史锚点。
4. 人物关系、人物纪要、出场人物、物品和场景索引由副 API 后台更新，但只有完整结构通过校验后才一次性提交。
5. 发送给主模型时，旧原文先按“最近原文窗口”裁剪，再注入一个顺序固定的六段系统记忆块。

默认六段顺序：

1. 人物关系。
2. 累计历史合并与其后的分段锚点。
3. 当前追踪角色的动态演变。
4. 最近上下文中名称命中的出场人物。
5. 重要道具、伏笔和内部梗。
6. 尚未进入锚点的逐楼摘要；只有开启“额外旧楼动态召回”后，才追加相关旧楼细节。

## 三、已修复的隐性问题

### 1. 长聊天提示词无限增长

旧版各块虽有局部裁剪，但最终系统块仍可能增长到较大体积。0.9.2 设置 32,000 字符总上限，并将关系、锚点、角色动态、人物、物品、未锚定摘要和动态召回分别限额。默认关闭额外旧楼召回，使楼层增长主要影响存储，不再线性扩大每次主模型输入。

### 2. 中文召回既慢又容易命中通用词

旧关键词切分会把长中文片段当成单个词，且候选评分容易被“当前、关系、随后”等通用剧情词抬高。新版使用受限的中文 n-gram、历史摘要分词缓存、查询词上限、候选上限和轻量 IDF 降权。Prompt Ready 阶段仍不等待网络向量；向量预取未完成时立即走本地关键词回退。

### 3. 删除中间楼层导致后续键整体漂移

没有原生消息 ID 的导入聊天曾以数组下标作为键。删除第 23 楼会让之后所有楼层换键，引发摘要、锚点、合并和关系回滚连锁失效。新版给消息写入插件稳定键，并把旧 Godlog、锚点、合并、消息召回记录和关系历史同步迁移到新键。

### 4. 隐藏旧消息后可见楼层缓存不刷新

隐藏状态可能改变中间消息，但旧缓存只观察聊天长度和尾消息。新版在批量隐藏/恢复后显式清空运行时楼层缓存，避免面板和发送前裁剪继续使用旧可见集合。

### 5. 副模型返回半截表格仍被标记为成功

增量人物索引过去可能先部分覆盖现有表，再把该楼标记为已处理。新版先在脱离当前数据的候选对象中检查：当前时间、当前地点、五个必需分区、Markdown 表骨架、固定关系名单的完整性、重复/额外行。任何一项失败都不提交，并把索引标为待安全重建。

### 6. 关系表显示“待重建”但仍发给主模型

旧逻辑在面板中声明脏关系不会注入，实际人物关系块没有检查 `relationshipDirty`。新版脏状态只发送重建占位，不发送旧关系内容。

### 7. 重复点击生成重复锚点或合并

锚点准备期、锚点写入期和累计合并期现在分别有互斥锁；锚点内部允许按既定顺序触发合并，但外部重复点击不会并发创建相同周期记录。

### 8. 切换聊天后旧请求写入新聊天

副 API 请求最长可持续 120 秒。旧版在等待期间切换聊天后，任务尾部仍可能调用当前聊天的保存、隐藏或刷新逻辑。新版为每个任务捕获聊天对象、metadata、存储 ID 和上下文代次；上下文变化后直接丢弃旧结果，旧任务也不能清除新聊天的任务锁。

### 9. Prompt Ready 重复注入

同一个最终消息数组若被多个检查器或事件重复处理，旧版可能再次插入相同系统块。新版会识别并移除已存在的六段 Anchor Memory 块后再插入。

### 10. 注入深度 0 实际无效

`Number(value) || 4` 会把合法的 0 当成空值。新版用显式有限数判断，因此 0、1、4 等设置均按真实值工作。

### 11. 面板文案与真实逻辑不一致

新版不再把可调间隔写死成 15/100；明确“自动隐藏”只影响显示，而发送前原文裁剪是独立机制；明确人物名称匹配与旧楼动态召回是两个开关；明确向量 API 只服务于可选召回；明确关系行不会自动加入人物动态白名单。

## 四、性能边界

- 工作台关闭时，不重建完整卡片、表格和诊断视图。
- 流式 token 阶段只做节流的尾楼探测，不逐 token 扫描全聊天。
- Godlog、聊天行、消息映射和关键词集合均有缓存。
- 动态召回默认关闭；开启后扫描的是摘要而非完整旧正文，且查询词、候选数、输出条数和字符数均有限制。
- 最终主模型记忆块最多 32,000 字符。实际常见输入通常低于该上限；若所用模型上下文较小，可关闭重要物品持续注入或降低副模型锚点篇幅。

## 五、需要用户本机验证的项目

1. 当前 SillyTavern 版本是否发出 `CHAT_COMPLETION_PROMPT_READY`，以及最终 Prompt Inspector 中是否只出现一个“锚点记录”系统块。
2. 所用副 API 是否稳定遵守 Markdown 表结构；不遵守时插件会保留旧数据并显示待重建，而不会静默覆盖。
3. 第三方主题是否遮挡顶部入口；仍可从扩展区按钮或控制台 `anchorMemoryOpen()` 打开。
4. 升级旧聊天后，先观察一次记忆体检与召回预览，再继续生成。
