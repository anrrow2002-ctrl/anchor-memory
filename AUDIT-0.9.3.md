# Anchor Memory 0.9.3 审计结论

## 已处理的高风险项

### 长楼层性能

- 流式 token 期间只探测尾楼，不全量重扫；
- 可见楼层、Godlog 索引和关键词切分继续缓存；
- 面板关闭时不重建全部卡片；
- 主模型记忆改为 Token 预算，避免累计锚点无限挤占上下文；
- 向量本体禁止进入聊天 metadata。

### 跨聊天与并发

- 锚点、合并已有互斥；
- 每个后台提交继续校验聊天引用与 storageId；
- 修正 storageId 校验读取键：使用 `anchorMemory`，不再误读 `anchor_memory`；
- 切换聊天主动取消副 API、Embedding 和模型列表请求。

### 时间与实体一致性

- 当前时间从仍有效 Godlog 重建，不依赖单次状态表随意覆盖；
- 回忆/梦境不推进现实时间；
- 明显倒退不提交为当前现实时间；
- 物品与场景使用稳定键；
- 用户删除产生墓碑，AI 旧表不能复活；
- 用户手动重加可解除墓碑。

### 主模型注入

固定六段顺序不变。最终消息数组仍会：

- 删除旧 Anchor Memory 系统消息；
- 裁剪旧正文，仅保留最近 N 个 AI 回合；
- 清理泄漏的 Godlog；
- 按可用 Token 分配六段；
- 记录实际注入 Token、字符和来源数。

## 保留的设计边界

- 人物关系表只跟踪固定人物与玩家；
- NPC 与其他角色关系继续在人物库文字列中表达；
- 不增加服装/情绪/待办账本；
- 不加入本地向量推理；
- 不追求 Horae 的 RPG、通用表格、多语言和主题生态。

## 仍需真实环境观察

容器无法启动用户本机的 SillyTavern、主题、酒馆预设和其他扩展组合，因此以下仍需安装后确认：

- 用户当前 SillyTavern 版本的 `CHAT_COMPLETION_PROMPT_READY` payload 形状；
- 特定主题下手机面板按钮位置；
- 浏览器 IndexedDB 配额和隐私模式行为；
- 所用免费模型对 3000–4200 输出 Token 的稳定性；
- 真实 1000+ 楼聊天 metadata 保存耗时。

这些不影响静态语法与纯逻辑回归结论，但属于真实宿主集成范围。
