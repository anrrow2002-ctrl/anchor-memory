# Anchor Memory 0.9.9 Injection Guard

面向长篇 SillyTavern 角色扮演的分层锚点记忆插件。

## 0.9.9 注入双保险与一键总开关

- 不再把“存在 `CHAT_COMPLETION_PROMPT_READY` 事件常量”等同于“本轮一定会触发最终提示词事件”。生成拦截器会先写入一份后端无关的保底记忆；最终提示词事件触发时再删除旧块、等待本轮动态召回并插入最新记忆，避免特殊后端或生成路径漏注入，同时保证最终请求只有一份 Anchor Memory。
- 工作台标题栏与 SillyTavern 扩展设置项都新增“暂停插件 / 启动插件”总开关。暂停不卸载、不删除任何数据，只停止主模型注入、旧楼隐藏、逐楼摘要、锚点、累计合并、动态召回和后台 API 请求；重新启动后继续使用原记忆。
- 暂停时会中止本插件可追踪的在途请求、清空待执行队列与提示词残留，并恢复由 Anchor Memory 管理的隐藏楼层；切换聊天或刷新页面后仍保持暂停状态。
- 新增随机压力模拟：1,200 次提示词注入、700 组 100–360 回合记忆连续性、1,500 次召回时序、300 次暂停/恢复。详见 `TEST-REPORT-0.9.9.md`。

## 0.9.8 记忆断层保护

未进入 15 回合锚点的 Godlog 会先转成逐回合紧凑连续索引，避免第六段预算裁掉中间楼层；当个别逐楼摘要失败并离开最近原文窗口后，插件会注入受限保底原文，并允许分段锚点、累计合并继续按时间顺序推进。失败 Godlog 本身仍保持待补写，不会被伪装成已完成摘要。详见 `MEMORY-FLOW-SIMULATION-0.9.8.md`。

## 本版定位

本插件继续围绕既有预设工作，不扩张成 Horae 式通用世界状态管理器。核心链路保持为：

`逐楼 Godlog → 分段锚点 → 累计历史合并 → 人物关系/人物纪要/人物库/物品/场景 → 六段主模型记忆`

明确不加入：

- 任意人物对的人物关系网络；
- 服装、情绪、待办、未完成约定、在场状态等独立账本；
- 浏览器本地向量模型或 Web Worker 推理。

人物关系仍然是用户固定名单中的“人物 ↔ {{user}}”；NPC 与主角之间的关系继续记录在出场人物库中。向量召回继续使用硅基流动等 OpenAI-compatible Embedding API。



## 0.9.7 关键修复：警告点击关闭 + 楼层间隔热调整

- “逐楼摘要缺失”警告恢复 SillyTavern 常规交互：点击警告只关闭，不再误打开锚点书控制面板；同时保留右上角关闭按钮。
- 新安装默认层级改为 `15 / 75`：每 15 个有效 AI 回合生成分段锚点，每 75 个有效 AI 回合生成累计历史合并，能够整批衔接。
- 两个间隔均可在聊天进行中随时调整。新值从下一批尚未处理的逐楼摘要开始生效；已经生成的锚点与累计合并不会被拆毁，也不会重复调用 API。
- 若调整时正有任务运行，该任务会锁定启动时的间隔完成；任务结束后插件会按新值自动重新检查待处理楼层。
- 降低阈值后，只要现有摘要已经满足新阈值，就会立刻进入后台队列，不必等下一条 AI 回复。
- 旧锚点会记录其实际批次数量，界面与召回记录不再把历史 15 回合锚点误标成用户后来设置的新数值。
- 两项不整除仍然安全：累计合并边界会用逐楼摘要补齐，不会漏楼、重复注入或跨边界重复计算。

本次不提升聊天记忆数据版本；已有 Godlog、锚点、合并、关系表、人物、物品、场景和向量引用会原样保留。旧用户已经手动设置的 `100` 不会被强制改写；`75` 是新安装与缺省配置的新默认值。

## 0.9.6 关键修复：动态召回必须先于主模型请求

修复“主模型已经开始输出，面板里才出现动态召回”的时序与显示混淆：

- 用户消息写入后立即开始预取；在 `CHAT_COMPLETION_PROMPT_READY` 的最终请求数组阶段等待语义召回；
- 语义 Embedding 最多等待 1800ms，及时返回就用于本轮主请求，超时则在发送前立刻改用关键词召回；
- 后到的语义结果不会伪装成本轮已使用内容，界面会明确显示“未用于本轮”；
- 工作台的“拼接预览”和 Prompt Preview 改为无副作用，不再覆盖本轮真正发送给主模型的召回记录；
- 本轮状态会明确显示“正在生成前召回 / 已在主请求发送前注入”、召回方式、等待时间、命中数量；
- Prompt Inspector 的 dry-run 只生成预览，不再污染正式生成记录。

本版仍保留 0.9.5 的手机全屏修复，数据版本不变，已有记忆无需迁移。

## 0.9.5 关键修复：手机端全屏面板

修复部分 Android 手机和 WebView 把页面宽度报告为 800–1000 CSS px，导致旧版 `520px` 全屏规则没有触发、锚点书只占屏幕上半截的问题。

- 面板尺寸改为跟随 `visualViewport`，浏览器地址栏变化、横竖屏切换与软键盘弹出时会重新同步；
- 手机和平板（含触屏设备与 1024px 内布局）统一使用真正的全屏面板；
- 打开面板时锁定底层聊天滚动，关闭后恢复；
- 12 个标签页改为单行横向滑动，不再挤成多行占掉半屏；
- 表单使用 16px 字号避免移动端聚焦缩放，并加入安全区、惯性滚动和横向表格滚动。

本版不改变记忆数据结构，已有逐楼摘要、锚点、人物、物品、场景、关系表和档案都会原样保留。

## 0.9.4 关键修复：已完成摘要锁定

逐楼摘要一旦成功生成并保存，就不再因为同一楼层后续发生以下变化而被自动判定为“缺失/待刷新”：

- 主模型记忆提示词被记录到该楼；
- SillyTavern 后补 `send_date`、消息 ID 或其他宿主元数据；
- 楼层重新渲染、隐藏/恢复、工具调用内容写入；
- 其他扩展补充显示数据；
- 用户编辑或切换该楼正文。

插件会保留原摘要、继续用于锚点和主模型记忆，并在展开区提示“摘要已保存并锁定”。只有用户主动点击“重跑本楼摘要”或手动编辑并保存摘要时，才会替换旧摘要并回滚、重建相关派生锚点。手动重跑采用事务提交：新摘要失败或生成期间正文再次变化时，旧摘要继续生效。

楼层身份也改为始终使用插件自有稳定键，不再在“临时键 / send_date / message_id”之间切换。升级时会把已有摘要、锚点、合并、关系历史和注入记录原位迁移到稳定键。

## 0.9.3 主要变化

### 1. Token 自适应记忆预算

不再依赖固定 32000 字符总裁剪。发送前根据本次上下文长度、最近正文和回复预留空间计算锚点书可用 Token，并按优先级分配：

1. 累计历史与分段锚点；
2. 人物关系；
3. 未锚定近期摘要；
4. 主角动态；
5. 物品；
6. 命中的 NPC。

默认上限 8000 Token，预留 1400 Token，均可在设置页调整。上下文已接近满载时，记忆块会继续缩小；不足以安全注入时会跳过，而不是挤爆请求。

### 2. 请求主动取消

副 API、Embedding 和模型列表请求统一进入请求注册表。切换聊天、刷新/关闭页面时会主动 `abort()`，旧任务既不会写入新聊天，也不会继续浪费连接和额度。

### 3. 向量存储安全降级

向量仍使用远程 Embedding API，但向量本体只存 IndexedDB。IndexedDB 不可用时：

- 自动改用关键词召回；
- 不把浮点数组回写聊天 metadata；
- 旧版 metadata 向量会被清除，可在存储恢复后重新构建。

### 4. 剧情时间连续性

从有效 Godlog 的 `<Time>` 重建当前现实时间：

- 回忆、梦境、假设和转述不会覆盖现实时间；
- “次日/翌日/第二天”等会向前推进；
- 日期后只写“下午/23:00”时会继承日期；
- 无依据的时间倒退会进入体检和场景页提示，并保留上一有效现实时间；
- 用户手动修正时间/地点后，后续楼层从该基线继续。

### 5. 物品与场景稳定实体层

界面和发送格式仍使用原有 Markdown 表格，但 metadata 中新增稳定实体键：

- 同名条目拥有稳定 ID；
- 同一场景按地点合并；
- 手动删除会建立墓碑，旧摘要不能自动复活；
- 用户重新手动添加时可解除墓碑；
- 不把服装、情绪和待办扩展成额外状态账本。

### 6. 工程与兼容性

- 首批纯逻辑拆分到 `core/runtime-controls.js`、`core/time-engine.js`、`core/entity-ledger.js`；
- 提供只读 `window.AnchorMemory` API；
- 增强窄屏/手机面板布局；
- 数据版本升级到 10，旧聊天自动迁移；
- 保留 SillyTavern 全局上下文优先、旧版动态导入回退的兼容策略。

## 安装

将整个 `anchor-memory-0.9.7-INTERVAL-TOAST-FIX` 文件夹作为 SillyTavern 第三方扩展安装。覆盖升级前建议先在“工具”页导出当前记忆 JSON。

升级后首次打开旧聊天会自动：

1. 建立时间轴状态；
2. 为现有物品与场景建立稳定实体键；
3. 迁移或清除旧 metadata 向量；
4. 保留已有 Godlog、锚点、合并、人物关系和索引。
