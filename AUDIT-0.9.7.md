# Anchor Memory 0.9.7 楼层间隔与警告交互审计

## 发现的问题

1. 缺失摘要警告为 toastr 配置了 `onclick`，点击后强制打开工作台并切换到“逐楼摘要”，覆盖了 SillyTavern 常规的点击关闭行为。
2. 新安装累计合并默认值仍为 100，与默认分段锚点 15 不整除，虽然底层能够用逐楼摘要补齐边界，但默认层级不够直观。
3. 设置页修改间隔只写入配置，没有主动重新检查已有待处理摘要；降低阈值后可能要等下一次消息事件才继续处理。
4. 生成中的锚点在等待摘要补写后才读取设置对象，玩家在任务中途修改数值可能改变当前批次边界。
5. 历史锚点在部分界面使用“当前设置”标注，修改间隔后旧的 15 回合锚点可能被误显示为新间隔。

## 修复结果

- 删除警告 toast 的工作台点击回调，明确启用 `tapToDismiss` 与关闭按钮。
- 默认层级改为 15 / 75；已有用户配置不强制覆盖。
- 间隔输入统一归一化到 UI 允许范围，并在修改后立即请求后台重新检查。
- 若摘要、锚点、合并或队列正在运行，则设置变更延迟到任务结束后重新检查，不丢失变更。
- 锚点与合并在任务开始时锁定间隔，并持久保存 `intervalUsed`、`batchSize` / `cycleSize`。
- 历史显示依据每条记录的实际来源数量，不再依赖当前设置。
- 已生成的派生记忆不自动拆毁；新值从下一批未处理摘要开始生效。
